# 社区报修派单系统

一个完整的社区报修工单管理平台，支持从居民报修到工单关闭的全流程管理，覆盖居民、调度员、维修人员、管理员四种角色。

## 🚀 快速开始

### 环境要求
- Node.js >= 18
- npm >= 9

### 安装依赖

```bash
npm install
```

### 启动开发服务（前后端同时启动）

```bash
npm run dev
```

- 前端地址: http://localhost:5173
- 后端地址: http://localhost:3000
- 健康检查: http://localhost:3000/api/health

### 构建生产版本

```bash
npm run build
npm start
```

## 📋 功能清单

### 核心功能
- ✅ **居民报修**: 填写标题、类别、地址、描述提交报修
- ✅ **调度派工**: 调度员选择维修人员派发工单
- ✅ **维修处理**: 维修人员填写处理结果
- ✅ **关闭工单**: 维修完成后关闭工单
- ✅ **退回补充**: 信息不全时退回给居民补充
- ✅ **管理员重开**: 已关闭的工单可由管理员重新打开

### 系统特性
- ✅ **角色权限**: 四种角色严格权限隔离
- ✅ **离线草稿**: localStorage 保存，刷新/重启浏览器可恢复
- ✅ **派工冲突检测**: 基于版本号乐观锁，409 冲突提示
- ✅ **操作日志**: 所有关键操作全程记录，可审计
- ✅ **CSV 导出**: 管理员可导出全部工单数据
- ✅ **数据持久化**: JSON 文件存储，服务重启数据不丢失

## 📂 目录结构

```
├── api/                     # 后端 (Express + TypeScript)
│   ├── db/database.ts       # JSON 文件数据库
│   ├── middleware/auth.ts   # 鉴权与权限中间件
│   ├── routes/              # API 路由 (auth/tickets/users/logs/export)
│   ├── services/            # 业务逻辑层 (工单/日志服务)
│   ├── types/               # 后端类型定义
│   ├── app.ts               # Express 应用
│   └── server.ts            # HTTP 服务器入口
├── src/                     # 前端 (React + TypeScript)
│   ├── components/          # 公共组件 (布局/弹窗/详情等)
│   ├── pages/               # 页面组件 (按角色分子目录)
│   │   ├── resident/        # 居民端 4 个页面
│   │   ├── dispatcher/      # 调度员端 2 个页面
│   │   ├── repair/          # 维修员端 2 个页面
│   │   ├── admin/           # 管理员端 4 个页面
│   │   └── Login.tsx        # 登录页
│   ├── store/               # Zustand 状态管理 (Auth + Draft)
│   ├── types/               # 前端类型定义
│   ├── utils/               # 工具函数 (API 封装/格式化)
│   ├── App.tsx              # 路由配置
│   └── main.tsx             # React 入口
├── shared/types.ts          # 前后端共享类型
├── data/db.json             # 数据库文件 (自动生成)
├── exports/                 # CSV 导出目录 (自动生成)
└── package.json
```

## 🔌 API 参考

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | /api/auth/login | 登录获取 Token | 公开 |
| POST | /api/auth/logout | 登出 | 登录用户 |
| GET | /api/users?role=repair | 获取维修人员列表 | 调度员 |
| POST | /api/tickets | 创建工单 | 居民 |
| GET | /api/tickets | 获取工单列表（按角色过滤） | 登录用户 |
| GET | /api/tickets/:id | 获取工单详情 | 按角色权限 |
| PUT | /api/tickets/:id/assign | 派工（version 乐观锁，冲突返回409） | 调度员 |
| PUT | /api/tickets/:id/return | 退回工单补充信息 | 调度员 |
| PUT | /api/tickets/:id/resubmit | 退回后重新提交 | 居民 |
| PUT | /api/tickets/:id/process | 填写处理结果 | 维修人员 |
| PUT | /api/tickets/:id/close | 关闭工单（校验处理结果） | 维修人员 |
| PUT | /api/tickets/:id/reopen | 重开已关闭工单 | 管理员 |
| GET | /api/logs | 操作日志列表 | 管理员 |
| GET | /api/export/tickets | 导出工单 CSV | 管理员 |

## 🛠️ 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 18 + TypeScript + Vite | SPA 单页应用 |
| 路由 | React Router v6 | 路由守卫、权限控制 |
| 状态 | Zustand | Auth 状态 + 草稿状态 |
| 样式 | TailwindCSS 3 | 原子化 CSS |
| 图标 | Lucide React | 轻量图标库 |
| 后端 | Express 4 + TypeScript | RESTful API |
| 存储 | JSON 文件 (data/db.json) | 轻量持久化，无需额外数据库 |
| 鉴权 | Bearer Token | 基于 Token 的会话管理 |

## ⚠️ 注意事项

1. **派工冲突测试** 需要同时打开两个不同的浏览器窗口（或无痕模式）分别登录两个调度员账号
2. **草稿自动保存** 间隔 30 秒，也可随时点击「保存草稿」按钮手动保存
3. **删除数据库**：如需重置所有数据，删除 `data/db.json` 文件后重启服务即可（会自动初始化样例数据）
4. **CSV 导出文件** 同时保存到服务端 `exports/` 目录，文件名带时间戳
